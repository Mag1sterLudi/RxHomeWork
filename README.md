## Структура программы

## Система построена на паттерне Observer и состоит из 4 основных компонентов:

**Основные компоненты:**

1. Observable - источник данных с операторами преобразования
2. Observer - получатель событий (onNext, onError, onComplete)
3. Schedulers - управление потоками выполнения
4. Disposable - управление жизненным циклом подписок

**Основные связи компонентов:**

1. Observable создает потоки данных и отправляет их Observer
2. Operators преобразуют данные между Observable и Observer
3. Schedulers управляют потоками выполнения для всех операций
4. Disposable контролирует жизненный цикл подписки Observer на Observable

**Ключевые особенности:**

- Неизменяемость объектов 
- Ленивые вычисления
- Композиция операторов
- Безопасность типов
- Реализованные операторы 

map(Function mapper)

**Преобразует каждый элемент потока**

.map(x -> x * 2)  // Удваивает числа
filter(Predicate predicate)

**Фильтрует элементы по условию**

.filter(x -> x > 5)  // Оставляет числа > 5
flatMap(Function mapper)

**Преобразует элементы в Observable и объединяет**

.flatMap(x -> Observable.create(...))  // Работа с вложенными потоками

## Система планировщиков (Schedulers)
**IOThreadScheduler**
Тип: CachedThreadPool
Применение - файловые операции, сетевые запросы, БД
Особенности - создает потоки по требованию, переиспользует существующие

**ComputationScheduler**
Тип: FixedThreadPool(2)
Применение - вычисления, алгоритмы
Особенности - фиксированное количество потоков, без блокировок

**SingleThreadScheduler**
Тип: SingleThreadExecutor
Применение - последовательная обработка, синхронизация
Особенности - один поток, гарантия порядка выполнения

**Принципы работы:**

subscribeOn(scheduler) - определяет поток источника данных

.subscribeOn(Scheduler.io()) - источник в IO потоке

observeOn(scheduler) - определяет поток обработки событий

.observeOn(Scheduler.computation()) - обработка в computation потоке

**Различия планировщиков:**

- IO - для блокирующих операций, динамический пул
- Computation - для вычислений, фиксированный пул
- Single - для последовательности, один поток

## Обработка ошибок и исключений
Механизм обработки ошибок реализован через метод onError(Throwable error) — терминальное событие в реактивном потоке. 
Он вызывается при возникновении ошибки на любом этапе обработки данных: в источнике, операторе или при выполнении задачи в планировщике.

**Основны работы onError():**
После вызова onError() поток завершается: последующие вызовы onNext() игнорируются.
Ошибка доставляется подписчику (Observer), если тот реализовал метод onError().
Если подписчик не обрабатывает ошибки (не реализует onError()), возникает исключение OnErrorNotImplementedException.
Гарантируется, что ошибка будет обработана ровно один раз — даже при множественных ошибках в цепочке операторов.
Выполняется автоматическая очистка ресурсов: вызывается dispose() для всех активных Disposable.

**Что происходит при исключении в операторе (например, в map или filter):**
Исключение перехватывается.
Поток немедленно прерывается — новые элементы не генерируются.
Вызывается onError() у всех подписанных Observer.
Выполняется очистка ресурсов: освобождаются все активные подписки через механизм Disposable.
Планировщики продолжают работать (если не вызван shutdown()).

## Примеры использования
**Простая обработка данных:**
```
Observable.create(source)
.filter(x -> x % 2 == 0) // Четные числа
.map(x -> "Число: " + x) // В строки
.subscribe(observer);
```
**Асинхронная обработка:**
```
Observable.create(fileReader)
.subscribeOn(Scheduler.io()) // Чтение в IO
.map(data -> processData(data)) // Обработка
.observeOn(Scheduler.computation()) // Вычисления
.subscribe(observer);
```
**Управление подписками:**
```
Disposable subscription = observable.subscribe(observer);
subscription.dispose(); // отмена
```

**Обработка вложенных потоков:**
```
Observable.create(source)
.flatMap(x -> Observable.create(...))  // Вложенные операции
.subscribe(observer);
```
